Missions

Missions are sets of tasks users complete during a campaign to earn rewards — from QR scans to daily logins.

ℹ️
Missions live inside Campaigns. Each campaign can have one Event Mission containing multiple tasks. Missions can also run standalone (no campaign) for always-on engagement like weekly login challenges.

The overall wrapper that holds all tasks — configure the schedule, audience, and completion rewards.

Mission Item (Task) Settings
✅
Mission Items
Each task inside an event mission. You control how it is completed, what it rewards, and the order users must do them.
SettingWhat it means
Name & DescriptionWhat the user has to do — shown in English and Chinese
Mission TypeHow the task is completed — QR Scan, Product Purchase, Custom Action, or Manual
Verification TypeAuto = instant completion · Manual = reward is withheld until a staff member approves in the Verification Queue
Is OptionalIf ticked, this task does not block completing the overall mission
Is ProgressiveFor "do this X times" tasks. Set a target count and reward distribution method.
Target CountHow many times the user must perform the action (e.g. 3 for "buy 3 vouchers")
Reward DistributionOn Complete (all at 100%) · Per Step · At Milestones
Points RewardFun Points given on task (or step) completion
Draw Chances RewardLucky draw entries given on task (or step) completion
PrerequisitesOther tasks that must be done first — hold Ctrl/Cmd to select multiple. Users cannot attempt a task until all its prerequisites are completed.
Display OrderThe order tasks appear to users in the app
⚙️ How Progressive Tasks Work
  • Turn on Is Progressive and set a Target Count.
  • The system tracks user progress (e.g. 1/3, 2/3, 3/3) and shows it in the app.
  • On Complete (all at 100%) — reward only when the user hits the full count.
  • Per Step — reward given every time the user makes one unit of progress.
  • At Milestones — reward at specific checkpoints you define.

⚙️ Key settings

  • Prerequisites chain Day 1 → Day 2 → ... → Day 7 to encourage daily return.
  • Day 7 requires all 6 previous days — users must complete the full chain for the bonus.
  • Not linked to a campaign — this is a standalone always-on mission.
Verification & Tracking
🔍
Mission Verification Queue
When a task has Verification Type: Manual, user submissions appear here as Pending. Staff review and approve or reject each one.
1
User submits

Their task completion appears in the queue as Pending.

2
Staff reviews

Check the proof or details provided.

3
Approve or Reject

Approved = task complete, reward given automatically. Rejected = no reward, user notified.

📈
Mission Tracker
A reporting view of who completed which task, when, and whether it was verified. Use this to investigate support queries like "Why didn't I get my reward?"
