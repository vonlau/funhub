Campaigns

Create and manage time-limited promotions — with a lucky draw, collectibles, and/or event missions — all in one place.

ℹ️
What is a Campaign? A campaign is the container for one promotion — e.g. CNY 2026 Lucky Draw or Collect & Win. Inside it you can set up a lucky draw (reward pool), a collectible set, and/or a mission with tasks. Everything runs under the same schedule and visibility settings.
Campaign Setup
🚀
Campaign Setup — Step by Step
New to campaigns? Follow these 5 steps to get one live from scratch.
1
Create the Campaign

Go to Campaigns → Create. Set the name, dates, and enable the features you need (Reward Pool, Collectibles, Event Mission). Save — the campaign is in draft, not live yet.

2
Manage Reward Pool → Create Reward Pool

Open the campaign → Campaign Collectibles → Manage Reward Pool → Create Reward Pool. Give it a name, tick Is Default Pool, and set Fun Points Per Draw if users can spend points to draw (leave empty to disable).

3
Add Items to the Pool

Inside the pool, click Add Item for each prize. Set Reward Type, Percentage, and Quantity.

Reward TypeWhat to prepare beforehand
Reward ComponentPre-create under Points & Rewards → Reward Components
Promotion CodePre-create under Points & Rewards → Promotion Code Groups
Better Luck Next TimeNo setup needed — always add one with unlimited quantity as fallback
⚠️
Pre-create reward components and promo codes first. You cannot create them from inside the pool. Set them up under Points & Rewards before coming back here.
4
Create the Mission

Campaign → Event Mission → Create Mission. Set name, visibility, repeat type, and Completion Draw Chances (bonus draw entries when a user finishes all tasks).

5
Add Mission Items (Tasks)

Inside the mission, click Add Mission Item for each task.

SettingWhat to set
Mission TypeThe action — QR Scan, Product Purchase, Custom Action, Manual
Is ProgressiveOn if the task must be done multiple times — set Target Count
Draw Chances RewardDraw entries given when the task is completed
Points RewardFun Points given on task completion
Total QuantityCap on total completions across all users (leave empty = unlimited)
PrerequisitesTasks that must be done before this one
💡
Ready to go live? Once all pool items total 100% and at least one mission item is active, go back to the campaign and set it to Active + Visible.

Reward Pool (Lucky Draw)
🎰
Manage Reward Pool
A campaign can have one reward pool — the "bag" of prizes users draw from. You create the pool first, then add items to it.
⚙️ How the Lucky Draw Works
  • Users spend draw chances (earned from missions) or Fun Points to draw from the pool.
  • The system picks one outcome based on each item's win percentage.
  • Users will always use mission draw chances first, then Fun Points if available.
  • If no real prize is available, the user gets the Better Luck Next Time fallback.
  • Once users have participated, the pool cannot be changed.
➕
Create Reward Pool
Set up the lucky draw container for a campaign.
FieldWhat it means
NameInternal name for this pool (e.g. "Night Party Pool")
DescriptionOptional internal notes
Is Default PoolMark this as the campaign's main pool (only one default allowed per campaign)
Fun Points Per DrawOptional — the number of Fun Points a user spends to do one draw. Leave empty to disable Fun Points draws. Users use mission draw chances first, then Fun Points.
Is ActiveWhether the pool is active
🎁
Add Item to Pool
Each item is one possible outcome in the draw. All active items' percentages must total 100%.
FieldWhat it means
Reward TypeWhat this prize is — choose from three types (see below)
PercentageWin chance for this outcome (0.00–100.00). All active items must total 100%.
Total Quantity AvailableStock limit for this prize. Leave empty for unlimited. Once stock runs out, this item is skipped in draws.
Quantity Given to UserHow many of this reward component the user receives when they win (default: 1)
Receive Once Per UserIf ticked, each user can only win this prize once in the campaign
Requires FunCard PurchaseIf ticked, this prize is only available to users who purchased a FunCard during the campaign period
Is ActiveToggle this item on or off without deleting it
⚠️
Percentages must total exactly 100%. All active pool items must add up to 100.00%. The system will block publishing if they do not. Inactive items are excluded from the total.
Reward Types
TypeWhen to use it
Reward ComponentA defined reward — points, a badge, or a voucher. Created under Points & Rewards → Reward Components.
Promotion CodeA code from a Promotion Code Group. Created under Points & Rewards → Promotion Code Groups.
Better Luck Next TimeThe fallback "no prize" outcome. Must have unlimited quantity so it never runs out. Always include one in every pool.
⚠️
Always add a Better Luck Next Time item. Without a fallback, the system has nothing to give users when other prizes run out or are ineligible. Set it to unlimited quantity.
Collectible Set
🃏
Collectible Set
Users collect items from lucky draws. When they have the full set, they win the completion reward.
⚙️ How Collectibles Work
  • You define the items users need to collect (e.g. Card A, Card B, Card C).
  • Users receive collectible items as prizes from the reward pool draws.
  • When all requirements are met, the system grants the completion reward automatically.
  • If Repeatable is on, users can complete the set more than once.
⚠️
Link collectible items to the reward pool. Collectible items must be set as prizes in the reward pool — users collect them by drawing from the pool. A collectible set with no matching pool items will never be completable.
SettingWhat it means
Name & DescriptionWhat this collectible set is called
RepeatableWhether users can complete the set multiple times
Completion RewardWhat users get when they finish — a reward component, badge, or promo code group
RequirementsList of items to collect and how many of each (usually 1 per card type)
Other Campaign Features
🏆
Lucky Draw Winners
After draws are run, see who won which prize. Export the list for prize fulfilment (e.g. physical prizes or vouchers).
📅
Reservations
Users request limited slots (e.g. event seats). Approve to confirm or Reject to decline.
⚠️
Add test accounts to Blacklist Seeder first. Go to Users → Blacklist Seeder before approving reservations so internal accounts don't take real slots.
🏅
Badges
Achievements awarded when users meet a set condition. Shown on profiles and can gate who sees an event mission.
🎁
Rewards Received
Full audit trail of every reward granted by this campaign — from draws, collectibles, or badges. Auto-recorded.
