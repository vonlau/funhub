Roles & Permissions

Control which admin staff can access which parts of the portal.

👤
Roles
Each role is a named set of permissions (e.g. "Content Editor", "Finance Viewer"). Assign a role to each staff member to control their access.
Example RoleTypical Permissions
Super AdminFull access to everything
Content EditorArticles, Categories, Tags only
Finance ViewerFinance Reports and Transactions — read only
Support StaffUsers, Approvals, Help Center
🔑
Permissions
Individual access flags for each section — View, Create, Edit, Delete. Permissions are grouped by module and assigned via roles.
👥
Staff Accounts
The list of admin portal users. Assign roles here. Deactivate accounts when staff leave.
⚠️
Principle of least access: Only give staff the permissions they need for their role. Avoid giving broad access to avoid accidental changes.
